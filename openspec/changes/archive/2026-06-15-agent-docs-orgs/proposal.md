## Почему

У Decartes сейчас есть минимальный README, но нет организованной документации, которая помогает coding agents и контрибьюторам быстро понять структуру библиотеки, команды разработки, публичную поверхность API и безопасный workflow для изменений. Добавление агент-ориентированной документации снижает время входа в проект и делает будущую реализацию меньше зависимой от разовых ручных осмотров репозитория.

## Что изменится

- Добавить структуру документации для агентов, которая объясняет назначение Decartes, устройство репозитория, команды разработки, ожидания по тестированию и результат сборки.
- На верхнем уровне задокументировать публичные экспорты и основные концепции canvas drawing engine, чтобы агенты могли делать ограниченные по области изменения без догадок о границах ответственности.
- Добавить правила о том, где должны жить будущие agent-facing заметки и как поддерживать их актуальность.
- Не менять runtime API, package metadata, конфигурацию сборки или поведение отрисовки.

## Capabilities

### New Capabilities
- `agent-docs-organization`: Определяет обязательную структуру и требования к содержанию агент-ориентированной проектной документации.

### Modified Capabilities
- Нет.

## Влияние

- Затронутая документация: `README.md` и/или новые документационные файлы в локальной директории документации репозитория.
- Затронутый процесс: будущие workflow coding agents и контрибьюторов должны использовать организованную документацию как первый источник ориентации в репозитории.
- Затронутый код: не ожидается; это изменение только документации.
- Зависимости: нет.
