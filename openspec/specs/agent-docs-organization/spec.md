# agent-docs-organization Specification

## Purpose

Определяет структуру и содержание агент-ориентированной документации Decartes.js, чтобы coding agents и контрибьюторы могли быстро ориентироваться в проекте перед изменениями.

## Requirements

### Requirement: Точка входа в документацию для агентов
Репозиторий SHALL предоставлять корневую точку входа в документацию для агентов, которая ориентирует coding agents и контрибьюторов перед редактированием файлов проекта.

#### Scenario: Агент открывает документацию репозитория
- **WHEN** агент или контрибьютор просматривает корень репозитория
- **THEN** он может найти корневой файл документации для агентов с руководством по пониманию и изменению проекта

#### Scenario: README раскрывает документацию для агентов
- **WHEN** читатель начинает с `README.md`
- **THEN** README содержит ссылку на точку входа в agent-oriented документацию

### Requirement: Покрытие проектной ориентации
Документация для агентов SHALL описывать текущее назначение проекта Decartes, структуру исходников, точки входа публичного API, команды разработки, команду тестов, команду сборки и релевантный OpenSpec workflow.

#### Scenario: Агент планирует изменение исходного кода
- **WHEN** агенту нужно понять, где реализовано поведение Decartes
- **THEN** документация указывает релевантные исходные файлы и экспортируемую поверхность API

#### Scenario: Агент проверяет изменение
- **WHEN** агенту нужно проверить изменения документации или исходного кода
- **THEN** документация указывает доступные проектные команды для тестов и сборки

### Requirement: Организация документации
Документация SHALL отделять стартовые инструкции для агентов от долгоживущих reference-деталей, чтобы будущие обновления можно было вносить в сфокусированном месте.

#### Scenario: Maintainer обновляет структуру проекта
- **WHEN** меняется структура исходников, экспортируемые модули, package scripts или OpenSpec workflow
- **THEN** документация указывает, какие agent-facing документы нужно проверить или обновить

#### Scenario: Читателю нужен подробный проектный контекст
- **WHEN** читателю нужно больше деталей, чем дает корневая точка входа для агентов
- **THEN** точка входа ссылается на вспомогательную документацию со сфокусированным проектным контекстом
